Coq of OCaml

Some demos of the current development version of coq-of-ocaml. Write at web [at] clarus [dot] me for more information. Work currently made at Nomadic Labs. We use it to verify the implementation of Michelson, the language to write smart-contracts in the crypto-currency Tezos. Contributors needed!

Examples

  • the kernel of Coq (30.000 lines)
  • Tezos (200.000 lines)

We import the purely functional part of OCaml to Coq. We aim to generate readable Coq files with no encodings. The semantics of the generated files may not be exactly the same as the sources. One should do manual reviews to assert that the generated files are a reasonable formalization of the sources. We always generate something, even in case of errors (which are explicitely notified).

We compile OCaml projects by pluging into Merlin. This means that if you are using Merlin you can run coq-of-ocaml. The method to verify the termination of generated fixpoints is praying. The package coq-typing-flags can disable termination checks in Coq. Similarly, matching on GADTs may or may not work. For modularity, we support modules without functors as namespaces, and first-class modules encoded into dependent records. We do not support functors. We generate axioms for .mli files as it may help formalizations.

We do not support yet cross-referencing between the generated Coq files.
